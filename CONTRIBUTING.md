# DracoDownloader 贡献指南 (Contributing Guide)

感谢您对 DracoDownloader 项目的关注！为了确保项目的健康发展并避免未来的版权纠纷，请您在提交任何代码前，务必仔细阅读并遵守以下规范。

## ️ 核心版权与知识产权声明（提交前必读）

1. **版权归属**：本项目的所有核心代码、设计及相关资产的版权，均归原作者（或项目维护团队）所有。
2. **贡献者授权**：当您向本项目提交 Pull Request (PR) 或任何形式的代码贡献时，即表示您同意将这部分代码的版权及所有相关知识产权完全转让给项目原作者/维护团队。
3. **贡献者名单**：作为对您贡献的感谢，所有被合并的 PR 作者将被列入项目的“贡献者名单（Contributors List）”中予以鸣谢。但请知悉，列入名单仅代表致谢，不代表您对该项目拥有版权或所有权。
4. **重要警告**：**如果您不同意上述版权归属原则，请勿提交任何 PR。**

## 1. 核心开发原则
这是该项目的“底线”，任何新增代码都必须严格遵守：
*   **零 GPL 依赖**：绝对不能引入 GPL 或 LGPL 许可证的第三方库（此类许可证具有传染性，会迫使整个项目开源）。
*   **纯自研优先**：在实现网络协议等核心功能时，优先自己编写代码，而不是直接引用外部库。
*   **无状态异步**：必须使用 Python 原生的 `asyncio` 异步架构，保持代码的无状态特性。
*   **跨平台兼容**：代码必须同时兼容 Windows 和 Unix（Linux/macOS）系统。

## 2. 代码与测试规范
*   **代码风格**：遵循 Python 官方的 PEP 8 规范；必须使用类型注解（Type Hints）；文档字符串（Docstrings）必须采用 **Google 风格**。
*   **测试要求**：使用 `pytest` 和 `pytest-asyncio` 框架。所有新功能都必须有对应的测试用例，并且每次提交前都要跑一遍全量测试。

## 3. 如何提交代码（PR 流程）
项目采用标准的 Git Flow 工作流：
1.  **Fork** 仓库到你自己的 GitHub 账号。
2.  拉取并创建**特性分支**（命名格式：`feature/你的功能名`）。
3.  编写代码并使用规范的**提交信息**（格式：`<type>: <description>`，例如 `feat: add amazing feature`）。
4.  **Push** 到你的远程分支。
5.  在 GitHub 上发起 **Pull Request (PR)**，并务必勾选下方的“提交前确认清单”。

## 4. 提交前确认清单 (Checklist)
请在发起 PR 时，确保你的描述中包含并勾选以下内容：
- [ ] 我确认我的代码符合项目的开发原则（无 GPL 依赖、符合 PEP 8 等）。
- [ ] **我已阅读并同意：将本次提交代码的版权归属项目原作者/维护团队，并知晓提交 PR 即代表同意加入贡献者名单。**
- [ ] 我确认本次提交的代码为原创，或已明确标注来源并符合开源协议要求。
- [ ] 所有新增功能均已通过 `pytest` 测试。

## 5.社区行为准则
*   保持开放和包容，尊重每一位贡献者。
*   讨论问题时只聚焦于**技术本身**，不进行人身攻击或情绪化表达。
